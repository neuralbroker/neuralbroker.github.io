neuDB

A tiny, file-based, AI-native database. Zero dependencies. Human-readable JSON. Semantic search built in. pip install neudb

code pypi architecture

Overview

neuDB is what I'm building now. A database engine for AI applications that don't need Postgres overhead — agents, prototypes, local tools. JSON tables on disk, cosine similarity search on embeddings, text search fallback when embeddings aren't available.

Architecture

JSON tables (users, sessions, messages, memories, tags)
  ↓
Table API (insert, list, search_text)
  ↓
Embedding store → cosine similarity search
  ↓
Agent loop: retrieve past messages → inject prompt → store turn
  ↓
(Optional) FastAPI HTTP API (Swagger at /docs)

Features

  • Zero core dependencies — just Python stdlib + JSON files
  • AI memory schema: users, sessions, messages, tags, memories
  • CLI (neudb.py) and library (import neudb)
  • Agent CLI for Ollama/OpenAI with long-term memory
  • FastAPI API layer with optional [api] extra

Tradeoffs

  • Why file-based JSON? Debuggable, portable, zero setup. Right for agents and prototypes, not for high-write production.
  • Why not SQLite? Wanted human-readable storage and embedding-first search without ORM overhead.
  • Why zero deps? Install friction kills adoption for small tools. One pip install and it works.

Status

v0.4.0 on PyPI. pytest + ruff CI. Next: test coverage, docs mirror on this site, publish workflow.
