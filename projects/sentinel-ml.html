SentinelML

Credit-card fraud detection API. 0.94 ROC-AUC holdout. Drift as a first-class concern, not a research project.

code postmortem architecture

Overview

RandomForest on 25 hand-engineered features. Docker Compose stack with PostgreSQL, Redis, MLflow, Prometheus, Grafana. The model part was easy — everything after shipping is what makes it work.

Key features

  • txn_velocity_24h — top feature by SHAP, permutation importance, and partial dependence
  • SMOTE on training set only — without it, recall on fraud cases drops to unusable
  • Drift detection — PSI, Kolmogorov-Smirnov, chi-squared on rolling 1k window
  • Model-versioned Redis cache — payload hash + version timestamp invalidates on retrain

Tradeoffs

  • Why RandomForest? Interpretable features, fast iteration. Next version: gradient-boosted with calibrated probabilities.
  • Why Redis cache? Optional optimization. API continues without it if Redis is down.
  • Why Docker Compose? Full observability stack from day one — not just the API container.
