Overview

BlitzKode is a fine-tuned Qwen2.5-1.5B-Instruct coding assistant with a full training pipeline, GGUF export, and FastAPI streaming server. Built because I wanted a coding assistant that worked on a plane — no GPU required at inference time.

Problem

Small coding models hallucinate API signatures. Large models need 24+ GB VRAM to fine-tune. I needed something that trains on consumer hardware and refuses confidently when it lacks verified context.

Pipeline

Dataset → SFT → Reward SFT → DPO → LoRA (r=32)
  → Merge (bfloat16) → GGUF Q8_0 (1.53 GB) → FastAPI + streaming

VRAM: full fine-tune blew past 24 GB. LoRA + gradient checkpointing brings active memory under 8 GB. Q8_0 GGUF runs on CPU with mmap — cold start under 2 seconds.

Guardrail: _SIGNATURE_QUERY_RE regex refuses signature/usage lookups without source code in the prompt. No LLM involved — it never gets worse.

Benchmarks

  • 510+ Hugging Face downloads (full GGUF + separate LoRA adapter)
  • VRAM: 24 GB → ~8 GB (training), ~3 GB (inference float16), 0 GPU (Q8_0 GGUF)
  • Eval suite is smoke-test level — executable unit-test scoring is planned

Tradeoffs

  • Why LoRA after DPO? Export flexibility. Validate full SFT/DPO checkpoint, then LoRA merge for deployment without a second full pass.
  • Why GGUF over ONNX? Ecosystem support for local inference (llama.cpp, Ollama). One artifact that runs everywhere.
  • Why regex guardrail over better prompts? Small models ignore prompt instructions under pressure. Hard refusal is the only layer I trust.
  • Why DuckDuckGo research? Fragile HTML parser, but loud failure mode. Off by default in rate-limited profile.

Lessons learned

Fine-tuning is mostly data work. My first SFT run had good loss curves and bad completions because the eval tested the wrong thing. Hand-audited data fixed it. DPO was anticlimactic — SFT data is the whole game. (full writeup)

Future work

  • Collapse 5 training scripts into single train.py --stage
  • Executable unit-test eval suite (not just smoke tests)
  • Live demo playground on this site
