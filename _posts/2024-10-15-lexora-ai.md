I've read a lot of long PDFs — research papers, technical reports, documentation — and I've gotten tired of manually searching through them. Ctrl+F only gets you so far when you don't know the exact phrase you're looking for.

So I built Lexora AI. Upload a PDF, ask a question in plain English, get an answer with citations showing exactly where the answer came from.

## How it works

The system uses RAG (Retrieval-Augmented Generation):

1. The PDF is chunked and embedded using a sentence transformer.
2. User queries are embedded and the most relevant chunks are retrieved.
3. The chunks are fed as context to an LLM, which generates the answer with citations.

Stack: LangChain for the orchestration, FastAPI for the API layer, Streamlit for the UI, and ChromaDB as the vector store.

## Why it matters

RAG is one of the most practical LLM patterns right now. It solves the hallucination problem — the model grounds its answers in actual documents — and it works with proprietary data without fine-tuning.

The code is on [GitHub](https://github.com/neuralbroker/lexora-ai).

## What I learned

The chunking strategy matters more than the embedding model. Too-small chunks lose context; too-large chunks dilute relevance. I landed on 512-character chunks with 128-character overlap, which worked well for research papers.
